Aviation Booking and Resource Management System
Product Requirements Document (PRD)
Document Information
	• Version: 2.1
	• Last Updated: December 26, 2024
	• System Type: Aviation-focused Progressive Web Application (PWA)
1. Overview
1.1 Product Purpose
A comprehensive Progressive Web Application (PWA) designed specifically for aviation organizations (flight schools, flying clubs, FBOs) to manage aircraft bookings, pilot certifications, resource scheduling, and compliance tracking. The system supports aircraft and general resource management, booking capabilities, payment processing, and aviation-specific compliance features.
1.2 Target Users
	• Flight instructors and pilots (members only)
	• Flight school administrators
	• Aircraft owners and operators
	• Aviation organization management
	• Passengers (non-members, added by members)
2. Core Features
2.1 Calendar and Booking Interface
	• Calendar Views 
		○ Monthly View: Traditional calendar grid showing all bookings across all resources
		○ Weekly View: 7-day horizontal timeline with hourly slots
		○ Daily View: Resource-column layout with individual aircraft/resource columns
		○ Agenda View: List format showing chronological bookings
	• Daily View Features 
		○ Resource Columns: Each aircraft/resource displayed as separate column
		○ Resource Filtering: Toggle visibility of specific aircraft/resources
		○ Horizontal Time Axis: Hourly time slots displayed horizontally
		○ Multi-Resource Booking: Drag bookings between resource columns
	• Drag-and-Drop Functionality 
		○ Booking Creation: Drag to create new bookings on any resource
		○ Booking Movement: Drag bookings between resources and time slots
		○ Time Modification: Drag booking edges to extend/shorten duration
		○ Resource Transfer: Drag bookings between aircraft columns
		○ Conflict Prevention: Visual warnings for scheduling conflicts
	• Booking Resolution 
		○ 30-minute interval booking slots (default, admin configurable)
		○ Prevention of double-booking with admin override capability
		○ Real-time calendar updates across all users
		○ Live conflict detection during drag operations
		○ Snap-to-grid functionality for precise time alignment

2.4 User Management and Membership System
2.4.1 Membership Requirements
	• Members Only Booking: No guest bookings allowed
	• Passenger Addition: Non-members can be added as passengers by members
	• Member Authentication: Firebase Authentication with Google Sign-In
2.4.2 Membership Tiers and Pricing
	• Membership Levels (Admin configurable): 
		○ Student Pilot
		○ Private Pilot
		○ Commercial Pilot
		○ Certified Flight Instructor (CFI)
		○ Aircraft Owner
		○ Custom membership types
	• Tiered Pricing: Different hourly rates per membership level per resource
	• Membership Benefits: Access levels, booking priorities, rate discounts
2.4.3 Pilot Certification Tracking
	• Currency Requirements 
		○ Biennial Flight Review (BFR) expiration dates
		○ Medical certificate expiration
		○ 90-day passenger currency
		○ Night currency requirements
		○ Instrument currency (6 approaches, holding, tracking)
	• Endorsements and Ratings 
		○ Aircraft type endorsements
		○ High-performance endorsements
		○ Complex aircraft endorsements
		○ Tailwheel endorsements
	• Automatic Restrictions: Prevent bookings when currency expired
2.4.4 User Roles and Permissions
	• Admin: Full system access and configuration
	• Instructor: Student management, endorsement authority, bookable as resource
	• Member: Standard booking and flight privileges
	• Maintenance: Aircraft status updates, maintenance scheduling
	• External User Invitations: Specific access levels for visiting pilots
	• Role-Based Resource Designation:
		○ CFI/CFII: Automatically designated as bookable resource
		○ DPE (Designated Pilot Examiner): Bookable resource for checkrides
		○ Ground Instructor: Bookable for ground training sessions
		○ Student Pilot: Can be included as resource in their own bookings
		○ Private/Commercial Pilot: Can be included as resource when booking
		○ Maintenance Personnel: Bookable for scheduled maintenance work

2.4.5 Advanced Role-Based Access Control System
	• Granular Permission System: Screen-level access control for all user interfaces
	• Custom Role Creation: Admin ability to create roles with specific permission sets
	• Predefined Roles with Specific Access:
		○ Administrator: Complete system access including admin dashboard, role management, system settings
		○ FlightLine Manager: Members, aircraft details, maintenance items, flight line status monitoring
		○ Mechanic: Aircraft status viewing, maintenance logbook entry modification and deletion
		○ Treasurer: All charges viewing, payment refund capabilities, charge status verification
		○ Pilot/Student/Instructor/Examiner: Aircraft status, booking calendars, member directory with messaging
	• Permission Categories:
		○ Core Access: Dashboard, Calendar, Profile, Messages
		○ Fleet Management: Aircraft viewing/editing, maintenance logs, compliance tracking
		○ User Management: Member directory, membership management, role administration
		○ Financial: Payment viewing/processing, refunds, billing management
		○ System Admin: Settings, reports, admin dashboard, system logs
	• Role Management Interface:
		○ Visual role creation and editing with checkbox-based permissions
		○ Role duplication for similar permission structures
		○ Badge color assignment for role identification
		○ Import/export roles for system migration
		○ Admin-only role deletion (except administrator role)

	• Membership Dues: Recurring monthly/annual payments

3. Technical Requirements
3.1 Technology Stack
Frontend
	• Next.js with TypeScript: React framework with server-side rendering, static generation, and type safety
	• Styling Framework:
		○ Tailwind CSS: Utility-first CSS framework for rapid, consistent styling
		○ shadcn/ui: Modern React component library built on Radix UI primitives
		○ Radix UI: Headless, accessible components for complex interactions
	• Icons: Material Symbols, Hero Icons, Lucide React for comprehensive icon coverage
	• Animation: Framer Motion for smooth, performant animations and transitions
	• Typography: Modern sans serif fonts (Inter, Geist, Manrope) for optimal readability
	• Calendar System Features:
		○ Multi-view calendar (Monthly, Weekly, Daily, Agenda)
		○ Resource-column daily view with filterable aircraft/resources
		○ Complete drag-and-drop functionality for booking management
		○ Real-time booking conflict detection and prevention
		○ Touch-compatible drag operations for mobile devices
		○ Snap-to-grid time alignment with configurable intervals
	• Progressive Web App: Mobile-responsive, offline capabilities
	• Modern UI/UX: Aviation-themed, intuitive design with Carolina Sky Blue color scheme
	• Mobile-First Design: Hamburger navigation in upper left, optimized for mobile devices
	• Admin Customizable: Brand name, colors, and logo can be configured by administrators
Backend
	• Node.js: Firebase Cloud Functions
	• Firestore: Primary database for all data storage
	• Firebase Authentication: User management and security
	• Firebase Email: Automated notification services
Integrations
	• Google Calendar API: One-way sync for all bookings and resources
	• n8n Workflows: Post-flight automation and data processing
	• Weather APIs: Optional METAR/TAF integration
	• Analytics Tools: Usage and performance monitoring
3.2 Performance Requirements
	• Support for 1,000+ simultaneous bookings
	• Real-time calendar updates
	• Sub-second response times for booking operations
	• Offline capability for critical booking functions
	• Cross-device synchronization
3.3 Security and Compliance Requirements
	• Aviation Data Protection: Secure pilot and aircraft information
	• Payment Security: PCI DSS compliance for payment processing
	• Role-Based Access Control: Granular permission system
	• Audit Trails: Complete booking and modification history
	• Data Backup: Automated daily backups with disaster recovery
